Phase 6: Booking Flow Architecture

5-step wizard on a single route with URL state sync

Route & Component Structure

Route: /[slug]/book
// Single route, client-side wizard
BookingPage (server component)
├── Fetches: lodges, seasons, booking rounds, member session
├── Validates: auth, financial status, round eligibility
└── Renders:

BookingWizard (client component — "use client")
├── BookingContext — wizard state + URL sync
├── StepIndicator — progress bar (1-5)
├── Step components (rendered by step number):
│
Step 1: SelectLodgeDates
Lodge picker + availability calendar + date range
→ calls validateBookingDates server action
│
Step 2: AddGuests
Add self + search/add other members
→ fetches member list (family first)
│
Step 3: SelectBeds
Room/bed map with availability status
→ fetches real-time bed availability
│
Step 4: ReviewPricing
Per-guest tariff breakdown, discounts, total
→ calls calculateBookingPrice server action
│
Step 5: Confirm
Summary + confirm button
→ calls createBooking server action
→ rate limited via Upstash Redis
└── BookingSuccess — confirmation with reference number

URL State Sync

URL params at each step
Step 1: /book?step=1
Step 2: /book?step=2&lodge=uuid&checkIn=2027-07-01&checkOut=2027-07-05&round=uuid
Step 3: /book?step=3&lodge=uuid&checkIn=2027-07-01&checkOut=2027-07-05&round=uuid
Step 4: /book?step=4&lodge=uuid&checkIn=2027-07-01&checkOut=2027-07-05&round=uuid
Step 5: /book?step=5&lodge=uuid&checkIn=2027-07-01&checkOut=2027-07-05&round=uuid

Guest & bed selections stay in React state only (not in URL)
On refresh at step 3+, wizard restores lodge/dates from URL, member re-selects guests/beds

Server Actions (new in Phase 6)

src/actions/bookings/
calculatePrice — tariff lookup per guest, weekday/weekend split, multi-night discounts
createBooking — SELECT FOR UPDATE, concurrency check, insert booking + guests + transaction
generateReference — org prefix + year + 4-char random alphanumeric
getAvailableBeds — rooms/beds for a lodge+date range, excluding already-booked
getBookableMembers — org members the current user can add as guests (family first)
schemas.ts — Zod schemas for booking input validation
queries.ts — booking list, detail, member's bookings
