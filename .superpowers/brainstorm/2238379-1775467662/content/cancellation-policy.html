How It Works

  • Tiers evaluated top-down — days counted from check-in date, first match wins
  • Fallback is always "no refund" — covers anything below the lowest tier
  • Preview table — shows human-readable summary so admins can verify
  • Admin override — when actually cancelling, admin can always change the calculated amount
  • Default policy — automatically applied to new bookings at creation time
  • Add/remove tiers — flexible number of tiers per policy
