Booking Creation: Concurrency & Data Flow

How the createBooking server action handles concurrent requests on opening day

Sequence: What happens when a member clicks "Confirm Booking"

createBooking server action — step by step
1. Rate limit check (Upstash Redis)
→ 5 submissions/min per member
→ If exceeded: return "Too many requests, try again shortly"

2. Auth + eligibility check
→ Verify session, membership, financial status
→ Verify member's class is allowed in this booking round

3. BEGIN TRANSACTION

4. SELECT FOR UPDATE on availability_cache rows
→ Lock the date range rows for this lodge
→ Other concurrent bookings WAIT here

5. Verify beds still available
→ Check each selected bed isn't already booked
→ If taken: ROLLBACK, return "Bed X is no longer available"

6. Re-validate booking rules
→ Season, round open, min/max nights, member limits
→ Server-side re-check (never trust client state)

7. Calculate final pricing
→ Per-guest: tariff lookup by membership class
→ Per-night: weekday vs weekend rate
→ Discounts: 5-night / 7-night per guest
→ Sum guest totals → booking total

8. Insert records
→ bookings row (status: PENDING or CONFIRMED)
→ booking_guests rows (per guest with bed, tariff snapshot)
→ transactions row (INVOICE, debit for total amount)

9. Update availability_cache
→ Increment bookedBeds for each date in range
→ Bump version (optimistic concurrency guard)

10. COMMIT

11. Return success
→ Booking reference (e.g. POLS-2027-7K3M)
→ Redirect to confirmation page

Concurrent Booking Scenario

Two members booking the same bed at the same time
Member A
1. Rate limit ✓
2. Auth check ✓
3. BEGIN TXN
4. SELECT FOR UPDATE → acquires lock
5. Bed available? ✓
6. Rules valid? ✓
7. Calculate price
8. Insert booking
9. Update cache
10. COMMIT → lock released
11. ✓ Success!
Member B
1. Rate limit ✓
2. Auth check ✓
3. BEGIN TXN
4. SELECT FOR UPDATE → WAITS...
(blocked until A commits)
4. Lock acquired (after A commits)
5. Bed available? ✗ — taken by A
6. ROLLBACK
11. "Bed X is no longer available"
→ Redirected to step 3 (bed selection)
