Phase 7: Payment Data Flow

End-to-end flow for the three Stripe Connect features

1. Stripe Connect Onboarding

Admin connects their club to Stripe
Admin Settings → clicks "Connect with Stripe" → Server Action → creates Express account → Stripe API
Stripe API → returns onboarding URL → Server Action → saves account ID to org → DB: organisations
Server Action → redirects admin → Stripe Onboarding → admin fills KYC → Return URL
Return URL → server action checks account status → DB: set onboarding_complete

2. Member Payment (Pay Invoice)

Member pays an outstanding booking invoice
Dashboard → clicks "Pay Now" → Server Action → creates Checkout Session → Stripe API
Checkout Session params: stripe_account: org.stripeConnectAccountId
application_fee_amount: applyBasisPoints(totalCents, 100)
line_items: [{booking ref, amount}]
metadata: {transactionId, bookingId, organisationId}
Server Action → returns checkout URL → Client → redirect to Stripe Checkout → Stripe
Stripe → payment succeeds → Webhook → checkout.session.completed → DB Update
DB updates: transaction.stripePaymentIntentId = pi_xxx
new transaction: type=PAYMENT, amount=totalCents
booking.balancePaidAt = now()

3. Webhook Handler

POST /api/webhooks/stripe
// Verify signature with STRIPE_WEBHOOK_SECRET
// Extract event type
checkout.session.completed →
1. Look up transaction by metadata.transactionId
2. Create PAYMENT transaction (stripePaymentIntentId = session.payment_intent)
3. Update booking payment timestamps
checkout.session.expired →
1. Log expiry (no DB changes needed)
Idempotency: Check if PAYMENT transaction with this payment_intent already exists before creating. Stripe can send webhooks multiple times.
