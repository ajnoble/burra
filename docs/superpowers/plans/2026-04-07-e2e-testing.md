# E2E Testing (Playwright) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Playwright E2E tests covering 6 critical user flows (login, booking, admin members, dashboard, admin bookings, org picker) that run against the production container on `localhost:3010`.

**Architecture:** Playwright runs headless Chromium against the already-running app. A setup project authenticates as 3 test roles (admin, officer, member) and saves browser state. Custom fixtures provide pre-authenticated `Page` objects to each test. CI runs E2E after deploy via SSH.

**Tech Stack:** Playwright, `@playwright/test`, Chromium, GitHub Actions

import { test as setup, expect } from "@playwright/test";

const accounts = [
  {
    email: "marek.kowalski@example.com",
    password: "testpass123",
    file: "e2e/.auth/admin.json",
  },
  {
    email: "anna.nowak@example.com",
    password: "testpass123",
    file: "e2e/.auth/officer.json",
  },
  {
    email: "katarzyna.wojcik@example.com",
    password: "testpass123",
    file: "e2e/.auth/member.json",
  },
];

for (const account of accounts) {
  setup(`authenticate as ${account.email}`, async ({ page }) => {
    await page.goto("/polski/login");
    await page.getByLabel("Email").fill(account.email);
    await page.getByLabel("Password").fill(account.password);
    await page.getByRole("button", { name: "Sign in" }).click();
    await page.waitForURL("**/dashboard");
    await expect(page.getByRole("heading", { name: "Dashboard" })).toBeVisible();
    await page.context().storageState({ path: account.file });
  });
}
```

import { test, expect } from "../fixtures/auth";

test.describe("Booking flow", () => {
  test("booking page loads with lodge and round", async ({ memberPage }) => {
    await memberPage.goto("/polski/book");
    await expect(memberPage.getByRole("heading", { name: "Book a Stay" })).toBeVisible();
    await expect(memberPage.getByText("Polski Lodge, Mt Buller")).toBeVisible();
  });

  test("booking round shows name not UUID", async ({ memberPage }) => {
    await memberPage.goto("/polski/book");
    // The round should show a human-readable name, not a UUID pattern
    const roundSection = memberPage.locator("text=Booking Round").locator("..");
    const uuidPattern = /[0-9a-f]{8}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{4}-[0-9a-f]{12}/;
    // Check that no UUID is displayed as the round name
    const roundText = await roundSection.textContent();
    expect(roundText).not.toMatch(uuidPattern);
    // Should show actual round names
    const hasRoundName =
      roundText?.includes("Member Priority Round") ||
      roundText?.includes("General Booking");
    expect(hasRoundName).toBe(true);
  });

  test("can select dates and proceed to add guests", async ({ memberPage }) => {
    await memberPage.goto("/polski/book");

    // Find a date in the calendar that's in the future and click it (check-in)
    const calendarDates = memberPage.locator("[data-date]");
    const dateCount = await calendarDates.count();

    if (dateCount === 0) {
      // Fallback: navigate to a month within the season and click dates by text
      // Navigate forward to find season dates
      const nextButton = memberPage.getByRole("button", { name: ">" });
      // Click forward until we find clickable dates within the season
      for (let i = 0; i < 3; i++) {
        await nextButton.click();
        await memberPage.waitForTimeout(500);
      }
    }

    // Select check-in by clicking a date cell
    // We look for visible date buttons/cells — the exact selector depends on the calendar
    const firstAvailableDate = memberPage.locator("button:has-text('10')").first();
    if (await firstAvailableDate.isVisible()) {
      await firstAvailableDate.click();
      // Select check-out
      const checkOutDate = memberPage.locator("button:has-text('14')").first();
      if (await checkOutDate.isVisible()) {
        await checkOutDate.click();
      }
    }

    // Verify the date summary appears
    await expect(memberPage.getByText("Check-in:")).toBeVisible();

    // The Next button should be enabled
    const nextButton = memberPage.getByRole("button", { name: "Next: Add Guests" });
    await expect(nextButton).toBeVisible();
  });

  test("next button validates before proceeding", async ({ memberPage }) => {
    await memberPage.goto("/polski/book");

    // Without selecting dates, the Next button should be disabled
    const nextButton = memberPage.getByRole("button", { name: "Next: Add Guests" });
    await expect(nextButton).toBeDisabled();
  });
});
```

    needs: deploy
    runs-on: ubuntu-latest
    steps:
      - name: Run E2E tests on VPS
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.VPS_HOST }}
          username: ${{ secrets.VPS_USER }}
          key: ${{ secrets.VPS_SSH_KEY }}
          command_timeout: 5m
          script: |
            cd /opt/snowgum
            npx playwright install chromium --with-deps 2>/dev/null || true
            npm run test:e2e 2>&1
      - name: Upload traces on failure
        if: failure()
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.VPS_HOST }}
          username: ${{ secrets.VPS_USER }}
          key: ${{ secrets.VPS_SSH_KEY }}
          script: |
            cd /opt/snowgum
            if [ -d e2e/test-results ]; then
              echo "E2E test results exist — check VPS at /opt/snowgum/e2e/test-results"

```bash

npm run test:e2e:ui       # Interactive Playwright UI
